# Task Trek — трекер задач «Внешний мозг»

Реализация v1 по ТЗ ([docs/SPEC.md](docs/SPEC.md)): задача выгружается из головы за секунды, а система сама решает, когда её показать. Планирование привязано к **типам дней** (офис / удалёнка / выходной), а не к дням недели.

## Запуск

```bash
npm install
npm run dev
```

- Приложение: http://localhost:5173
- API: http://localhost:3001/api

База (SQLite) создаётся автоматически в `server/data/tracker.db` с текущей ротацией: пн–ср офис, чт–пт удалёнка.

## Возможности v1

- **Сегодня** — план собирается автоматически при первом открытии за день: дедлайны → задачи «на этой неделе» с подходящим контекстом → назначенные на дату → добивка quick-задачами из inbox, всё в пределах ёмкости дня. Кнопка «Пересобрать» доступна всегда; выполненные задачи при пересборке не теряются.
- **Быстрый ввод** с любого экрана: `оплатить кредит @10.07 !` → дедлайн 10 июля + флаг «на этой неделе».
- **Inbox** — тап по задаче раскрывает разметку: домен, контекст дня, усилие, дедлайн (+ жёсткий), статусы.
- **Неделя** — 7 дней с типами, дедлайнами, назначенными задачами и счётчиком тренировок.
- **Бэклог** — someday-задачи с фильтром по доменам.
- **Настройки** — ёмкость дней, смена ротации (старые планы не пересобираются), Google Calendar, уведомления.

## Google Calendar (опционально)

1. Создай OAuth-клиент в [Google Cloud Console](https://console.cloud.google.com/apis/credentials) (тип «Web application», redirect URI `http://localhost:3001/api/gcal/callback`), включи Calendar API.
2. `cp .env.example server/.env` и заполни `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`.
3. Перезапусти сервер, в «Настройках» нажми «Подключить Google Calendar».

После подключения: события календаря учитываются в ёмкости дня и показываются на «Сегодня»/«Неделе»; задачи с жёстким дедлайном попадают событием на весь день в отдельный календарь **Tracker** (обновляются и удаляются вместе с задачей). Полного двустороннего синка нет — намеренно.

## API (главное)

| Метод | Путь | Что делает |
|---|---|---|
| `GET` | `/api/today` | план на сегодня (собирает при первом обращении) |
| `POST` | `/api/today/rebuild` | пересобрать план |
| `POST` | `/api/today/tasks` / `DELETE /api/today/tasks/:id` | ручное добавление/удаление из плана |
| `GET/POST` | `/api/tasks` | список / создание задач |
| `POST` | `/api/tasks/quick` | быстрый ввод `{ "text": "… @15.07 !" }` |
| `PATCH/DELETE` | `/api/tasks/:id` | правка / удаление |
| `POST` | `/api/tasks/:id/complete` · `/reopen` | выполнить / вернуть |
| `GET` | `/api/week` | обзор недели |
| `GET/POST` | `/api/rotations` | ротации (новая запись = смена графика) |
| `GET/PATCH` | `/api/day-types` | типы дней и ёмкость |

Авторизация: задай `API_TOKEN` в `server/.env` — тогда все запросы требуют `Authorization: Bearer <токен>` (v2-задел под MCP/агентов).

## Деплой на VPS (Docker)

```bash
rsync -az --exclude node_modules --exclude .git --exclude server/data . root@СЕРВЕР:/opt/tasktrek/
ssh root@СЕРВЕР
cd /opt/tasktrek
echo "SITE_ADDRESS=tasktrek.IP-ЧЕРЕЗ-ДЕФИСЫ.sslip.io" > .env   # или свой домен
cp .env.example server/.env   # заполнить API_TOKEN и GOOGLE_*
docker compose up -d --build
```

Три контейнера-состояния: `api` (Fastify), `web` (Caddy: автоматический HTTPS, статика клиента, прокси `/api`). База — в Docker-томе `tracker-data`. На iPhone: открыть сайт в Safari → «Поделиться» → «На экран “Домой”» — приложение станет PWA с иконкой.

## Структура

```
server/   Fastify + better-sqlite3 (вся бизнес-логика, включая сборку «Сегодня»)
client/   React + Vite SPA, мобильная вёрстка, PWA
deploy/   Caddyfile для продакшена
```
